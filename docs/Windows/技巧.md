### 传输大型数据集

传输数据集的时候，可以通过 Windows 网络文件夹共享

不需要有路由器，只需要两个电脑用网线连起来就行（当然有路由器更好）

两者之间会使用 APIPA（Automatic Private IP Addressing）来自动分配 IP 地址 ，没有网管，但是在同一个网段下 （比如我这里分配到 169.254.111.5 和 169.254.174.197 两个保留地址）

对方计算机，选一个文件夹，属性为共享，然后添加 Everyone 即可

然后通过”运行“，输入 `\\169.254.111.5`  访问对方资源，这里要求输入对方计算机的”凭证“

如果对方登录了 Microsoft 账号，则输入邮箱和邮箱密码。此时输入计算机的用户名、登录密码都是不行的。（因为默认你的网络和共享中心里，是设置了”有密码保护“）

↑ 这里的微软账号登录，不需要计算机连网

如果没登 Microsoft，抱歉不知道。

（输入 cmd 中`whoami`返回的结果试一试？，类似`desktop-b6mln8c\name`或者`name`？）

不想继续共享了，选高级共享，上面有一个复选框，取消勾选。

### 远程桌面

你应该这样来填写：

计算机：填 ip

用户名：（1）填 C 盘下那个用户名，即不带 desktop-xxx 的 （2）填邮箱的密码

密码：登陆了 Microsoft 账号的电脑，密码必须是邮箱密码，不是锁屏密码

在这一点上，有点和 linux 的 SSH 类似，username@hostname

### 权限提升

https://zhuanlan.zhihu.com/p/627827866

今天突然看到 windows 权限相关话题，因此准备了解一下

windows 上有四种级别的权限 TrustedInstaller SYSTEM Administrator 用户

级别依次递减

SYSTEM 权限，是可以获取到的，通过 PsExec.exe -s 能够提权。网上也还有很多种提权办法。

实际上 TrustedInstaller 也可以获取到，但是不能在上下文中运行程序，这个权限受限了。

Windows Vista 企图赋予用户两方面的好处，一方面是允许管理员在标准用户的上下文环境下执行大部分的进程，仅通过用户准许来提升他们的用户令牌的特权，另外，允许标准用户通过选择提升一个进程，使用管理员级别的身份来执行管理员的任务。

### Powershell 技巧

整理一下 Power Shell 的常见脚本，以及操作技巧

https://learn.microsoft.com/zh-cn/powershell/module/microsoft.powershell.management/copy-item?view=powershell-7.4

它 powershell 所有的命令列在左边了，并且是分类的

1、取 Hash

（在 powershell 里，怎么取 hash，都是同一个命令，算法在参数里，而 Linux 里，一般直接用 md5 命令，就是算法本身是命令）

```powershell
Get-FileHash .\a.txt
Get-FileHash .\a.txt -Algorithm MD5
```

默认是 SHA256

你打出 Get-H 就可以按 TAB 来补全，然后打出 Get-FileHash 就可以开始尝试按 TAB 给你选文件
