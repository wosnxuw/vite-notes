### Shell Bash Terminal

Terminal：终端相当于一个程序，一个窗口。比如 Win11 的那个新终端，它只是一个 Terminal，给用户操作。

Shell：是用户键入的命令的解释器，**本身并不具体指代哪一种**

Bash：Linux 下最常见的 Shell，也就是 Shell 的具体实现

Cmd、PowerShell：Windows 上的两个 Shell

Zsh：另一个 Unix 上的 Shell。MacOS 切换到 Zsh 上，使得越来越多的人用这个 Shell，很多发行版默认是 Bash，但是也给你安装了 Zsh。

Git Bash：Git for Windows，用 MSYS2 还是什么的，做了一个模拟兼容层，模拟 bash。所以它本事是一个 Shell

### 桌面环境 DE

**X Window System 或 Wayland**：这是图形显示的基础。X Window System（通常称为 X11）是传统的显示服务器，而 Wayland 是现代替代品。它们负责管理图形显示、输入设备等。

**桌面环境（Desktop Environment）**：提供完整的用户界面，包括窗口管理器、面板、菜单、文件管理器、终端等一系列应用程序和工具。常见的桌面环境有 GNOME、KDE、XFCE、LXDE 等。

**GTK**： 主要用于 GNOME、XFCE
**Qt**： 主要用于 KDE Plasma

GNOME，现代，简洁且高效

KDE，可自定义程度高

XFCE，轻量级

Ubuntu Gnome

XUbuntu XFCE

KUbuntu KDE

Debian 默认是 Gnome

CentOS Gnome

OpenSUSE KDE Plasma

### 防火墙

iptables Linux 内核里的防火墙，但是用起来麻烦。在老的发行版上，就是内核；在新的发行版上，内核是 nftables，iptables 则是一个假命令，兼容层，转给其它命令处理

Ubuntu/Debian (UFW)

RHEL/CentOS/Fedora (FirewallD)，firewalld 的 d 是 deamon，就是后台服务，管理的工具则是 firewall-cmd

反正先用看服务是否已经跑起来，然后 ss 看是否监听端口，最后用 ufw 打开端口

`ss -tlnp | grep :22`

`firewall-cmd --list-ports`

`firewall-cmd --add-service=ssh --permanent`

`firewall-cmd --reload`

### systemd

现代 Linux 系统的“第一个进程”（PID 1）

功能：系统初始化、服务管理 (nginx)、挂载文件系统、网络配置

systemctl 是 systemd 的控制工具

现在基本上我这种小白能说的上来的 Linux 都有 systemd。systemd 被人诟病就是说它比较臃肿，管理的事情太多

`systemctl enable xxx` 多数时候相当于设置这些的单元为开机自启动（但不会立即启动）
`systemctl disable xxx`
`systemctl is-enabled xxx` 返回 0 是有开机自启（但不一定当前就在工作）
`systemctl status xxx` 现在是否在跑
`systemctl start xxx` 跑起

### 变量

在 SHELL 中

定义变量：

```shell
MYVAR=a
```

右值的字符串可以用单引号，也可以用双引号，也可以不用引号

变量名建议大写，等号左右**建议无空格**

使用变量：

```
$MYVAR
${MYVAR}
```

### 环境变量

那么环境变量也是一种变量，叫做 PATH

查看环境变量

```shell
echo $PATH
```

Linux 环境变量分为：

永久的：需要用户修改相关的配置文件，变量永久生效

临时的：利用 export 命令，在当前终端下声明环境变量，关闭 Shell 终端失效

添加变量的姿势：

PATH=$PATH:（需要添加的环境变量） 进行拼接

PATH=（需要添加的环境变量） 将会覆盖之前的变量

### 文件夹

```shell
cd a

```

第一种系统会自己找，第二种明确告诉系统是一个目录，第三种明确在这里找，而不是找环境变量。

